## Issue #92 â€” Fix `tournament_stage = 'unknown'` for WC matches

### Root Cause

`matches_clean.csv` is built from the Kaggle raw CSV which has a `tournament` column but no `tournament_stage` column. `build_features.py` reads `getattr(row, "tournament_stage", "Unknown")` â€” since the column is absent, every row gets `"Unknown"`. The `demo_international_matches.csv` does have per-match stage data but `save_processed_matches` never derives or preserves the column.

A secondary bug: `normalize_tournament_stage("Quarterfinals")` returns `"quarterfinals"` (not in `STAGE_IMPORTANCE`), so quarterfinal importance is silently 0.

### Design

**New file:** `data/raw/wc_stage_lookup.csv`
Columns: `date, home_team, away_team, tournament_stage`
Contents: all 128 matches from WC 2018 and WC 2022 with canonical stage labels (`Group Stage`, `Round of 16`, `Quarter-final`, `Semi-final`, `Third Place`, `Final`). Team names must resolve via `resolve_team()`. WC 2026 matches are appended as they are played (see issue #93).

**`src/data/load_matches.py`**
- Add `_apply_stage_lookup(df) -> pd.DataFrame`: loads `wc_stage_lookup.csv`, left-joins on `(date, home_team, away_team)`, fills `tournament_stage` from the lookup where the existing value is absent or `"Unknown"`.
- Update `save_processed_matches` to call `_apply_stage_lookup` and always write `tournament_stage` as a column (defaulting to `"Unknown"` if neither input nor lookup has a value).

**`src/features/competition_weights.py`**
- Add `"quarterfinals": "quarterfinal"` to `_STAGE_NORM_MAP`.

**Verification**
- Re-run pipeline â†’ `python -m src.evaluation.backtest_world_cup`
- By-stage breakdown must show `group_stage`, `round_of_16`, `quarterfinal`, `semifinal`, `final` instead of all `unknown`.

## Issue #70 â€” Fix Ensemble Degeneration (LogReg/MLP weight = 0)

### Root Cause

SLSQP optimizer bounds are `[(0.0, 1.0)]` â€” no minimum per model. If XGBoost alone scores better val-set log-loss than any blend including LogReg/MLP, the optimizer correctly drives their weights to zero. LogReg and MLP may genuinely underperform because they were not retuned when the feature set grew.

### Design

**`src/models/train_ensemble.py`**
- Add `_diagnose_base_models(p_xgb, p_logreg, p_mlp, y_val)`: prints individual val-set log-loss for each model before weight optimization. Output is retained permanently so every ensemble retrain shows the per-model baseline.

**`src/models/train_logreg.py`**
- Expose `C` (inverse regularization strength) from `config.yaml` â†’ `model.logreg_C` (default `1.0`). Allows tuning without code changes.

**`src/evaluation/baselines.py` â€” `MLPModel`**
- Expose `hidden_layer_sizes` from `config.yaml` â†’ `model.mlp_hidden_layers` (default `[64, 32]`). Allows tuning without code changes.

**`configs/config.yaml`**
- Add `logreg_C: 1.0` under `model`.
- Add `mlp_hidden_layers: [64, 32]` under `model`.
- Add `min_model_weight: 0.05` under `model`.

**`src/models/train_ensemble.py` â€” `_optimize_per_class_weights`**
- Change `bounds` from `[(0.0, 1.0)]` to `[(cfg_min_w, 1.0)]` where `cfg_min_w = config["model"]["min_model_weight"]`. This prevents total zeroing while still letting the optimizer find the best blend within the floor.

**Retraining order (unchanged):** `train_xgb` â†’ `train_logreg` â†’ `train_draw_submodel` â†’ `train_ensemble`.

**Verification**
- Run `python -m src.models.train_ensemble` â€” diagnosis output must show non-trivial log-loss for all three models.
- Per-class weights must show no all-zero rows.
- Draw blend weight must be > 0.

## Issue #86 â€” Tournament-Specific XGBoost Model

### Design

**`src/models/train_xgb.py`**
- Add `--tournament-only` boolean flag.
- When set, after the `min_train_year` filter, further filter `df` to rows where `competition_weight >= cfg["model"]["tournament_model_min_weight"]`.
- This reuses 100% of existing training logic; only the data slice changes.

**`configs/config.yaml`**
- Add `tournament_model_min_weight: 3` under `model`.

**Training command:**
```bash
python -m src.models.train_xgb --model-name xgb_tournament --tournament-only
```

**`src/api/services.py`**
- On startup, attempt to load `xgb_tournament.joblib` after `ensemble.joblib`. If absent, log a warning and continue (general model used for all predictions).
- In `predict()`: compute `competition_weight` for the incoming request (already done). If `competition_weight >= cfg["model"]["tournament_model_min_weight"]` AND `xgb_tournament` is loaded, use `xgb_tournament` for prediction. Log which model was selected.
- No API schema changes â€” fully transparent to callers.

**Verification**
- Train tournament model and confirm `xgb_tournament.joblib` is created.
- Call `/predict` with `competition = "FIFA World Cup"` â†’ logs show tournament model used.
- Call `/predict` with `competition = "International Friendly"` â†’ logs show general model used.
